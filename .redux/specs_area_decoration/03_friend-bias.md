## Summary
This section implements (1) the "friend floor" proximity system: a per-(friendKey, z-floor) BFS distance field over the bounding rectangle of collected tiles (8-connected, unobstructed, so values equal Chebyshev distance to the nearest friend-ground tile), plus the density-bias blend formula density*( (1-chance) + chance*proximity^exponent ); (2) candidate tile collection from the map area (with cluster-rule and friend-ground exceptions to the blocked-tile filter, and groundId=0 sentinel for ground-less tiles when cluster rules exist); and (3) the two placement gates: checkSpacing (Chebyshev or Manhattan distance against already-previewed items via a spatial hash, with separate any-item and same-item minimums) and validateTilePlacement (tile must exist and not be blocking when skipBlockedTiles). None of these functions consume RNG, but their outputs (candidate list content/order, biased density) drive the RNG-roll sequence downstream, so they are determinism-critical.

=== applyFriendBias(rule, pos, baseDensity) — lines 955-987 ===
Returns a float density; no mutation, no RNG. Early returns of baseDensity UNCHANGED, in this order: (1) rule == null || rule->friendChance <= 0 || !rule->hasFriendFloor(); (2) from==0 || to==0 after the same from/to derivation as the cache builder (range wins over single id; swap if from>to); (3) distance = getFriendDistance(key, pos) < 0 (no cache layer / out of rect).
Otherwise: friendChance = rule->friendChance / 100.0f (float division), then clamped: if <0 → 0; else if >1 → 1.
proximity = 1.0f / (1.0f + float(distance)).   [d=0 → 1.0; d=1 → 0.5; d=2 → 0.3333…]
If rule->friendStrength > 0: exponent = 1.0f + (float(friendStrength) / 20.0f); proximity = pow(proximity, exponent).  [strength 20 → exp 2.0; strength 100 → exp 6.0; strength <= 0 → no pow at all.]
biasedDensity = baseDensity * proximity.
return baseDensity * (1.0f - friendChance) + biasedDensity * friendChance;   — i.e. result = baseDensity * ((1 - c) + c * proximity^e). On a friend tile (d=0) proximity is exactly 1 so result == baseDensity: the bias never INCREASES density above base; it only attenuates it with distance. All arithmetic is 32-bit float in C++.

## Porting notes

The getFriendDistance upper-bound test is half-open (x >= minX + width fails). 2) INTEGER DIVISION: queryRadius uses C++ truncating division (radius/8 + 1, pos/8). For non-negative map coordinates Lua's // (floor) is identical; if negative coords were ever possible they would diverge (and the C++ cell hash (uint64(cx)<<32)|uint64(cy) also sign-extends negative cy over the high bits — map coords are non-negative in RME so this never fires). 3) FLOAT PRECISION: applyFriendBias does everything in 32-bit float, including std::pow(float, float). Lua doubles will differ in low bits; since the result is compared against an RNG roll downstream, threshold-edge tiles can diverge per seed. For byte-faithful behavior, round intermediates through float precision (or accept rare divergence and document it). 4) MUTATION ORDER: boundsByZ is padded IN PLACE (only in real-map mode) BEFORE both the friend scan and the BFS — the BFS grid is the padded rect. Virtual mode never pads.
